GET /api/admin/users
  - Requires super_admin role (check deal_users.role for the authenticated clerk_id)
  - Return all users with: clerk_id, name, email, role, billing_status, billing_suspended_at, created_at

POST /api/admin/users/:clerkId/suspend
  - Requires super_admin
  - Set billing_status = 'suspended', billing_suspended_at = NOW()

  - Requires super_admin
  - Set billing_status = 'active', billing_suspended_at = NULL

GET /api/billing/status
  - Returns { billingStatus: 'active' | 'past_due' | 'suspended' } for the authenticated user

POST /api/billing/appeal
  - Body: { message: string }
  - No auth required (suspended users can't auth)... wait, they can still auth with Clerk, they just can't use the app. So require Clerk auth.
  - Insert into payment_appeals table with user's clerk_id, name, email from deal_users

GET /api/admin/appeals
  - Requires super_admin
  - Return all payment_appeals ordered by created_at DESC

POST /api/admin/appeals/:id/approve
  - Requires super_admin
  - Set appeal status = 'approved', reviewed_by = admin clerk_id, reviewed_at = NOW()
  - Also set the user's billing_status = 'active'
