CRITICAL NEW FEATURE: TRANSACTION STRUCTURE ROUTER
==================================================

The app must include a dedicated decision page and workflow engine for:

1. STOCK PURCHASE
2. ASSET PURCHASE

This is a required branching point in the acquisition process.

The product must not allow the user to continue into later close-readiness steps without first selecting and confirming the intended transaction structure.

This branch must be visually important and operationally intelligent.

Create a dedicated workflow step called:

“Transaction Structure”

This step should:
- educate the user on the practical difference between stock purchase and asset purchase
- clearly show operational consequences of each path
- store the selected structure as a deal-level setting
- dynamically change downstream workflows, tasks, required documents, and readiness logic

==================================================
STOCK PURCHASE PATH
==================================================

If user selects STOCK PURCHASE:
- continue under existing seller entity acquisition workflow
- preserve existing entity continuity logic
- keep stock transfer, corporate authority, and ownership transfer milestones active
- emphasize hidden liability review, tax exposure review, litigation review, corporate document review, and stock certificate transfer workflow
- trigger stock-purchase-specific due diligence requirements
- show continuity-sensitive items like:
  - stock ledger
  - corporate records
  - officer/director authorizations
  - shareholder approvals
  - legacy liabilities
  - tax exposure
  - existing licenses and change/update requirements

The UI should make stock purchase feel:
- fast
- continuity-driven
- potentially efficient
- but risk-sensitive

==================================================
ASSET PURCHASE PATH
==================================================

If user selects ASSET PURCHASE:
the app must require creation of a NEW BUYER ENTITY workflow before the user can advance to the rest of the acquisition sequence.

This is mandatory.

The app must create a dedicated step and page group called:

“New Entity Setup”

This module must function like a guided pre-close launch checklist for the acquisition vehicle.

The system must explain:
“In an asset purchase, assets are generally acquired into a new or designated buyer entity. Before proceeding, the buyer must establish the acquiring entity and complete required setup tasks.”

The workflow must include:
- entity type selection
- state of formation selection
- entity naming workflow
- entity availability/search step
- formation filing step
- EIN step
- bank account preparation step
- tax/account registration step
- licensing and permit setup planning step
- payroll/workers’ comp/insurance planning step
- operating agreement or corporate governance document step
- registered agent / address capture
- beneficial ownership / compliance placeholder if required
- new business license / automotive licensing workflow planning
- Day 1 legal operating readiness confirmation

==================================================
ASSET PURCHASE ENTITY SETUP WORKFLOW REQUIREMENTS
==================================================

The new entity setup must be a first-class product workflow, not a note or checklist buried in settings.

Build a full guided wizard with progress tracking.

Required sub-steps:

1. Entity Strategy
- choose LLC / corporation / other
- choose state
- name the acquiring entity
- note whether entity already exists or must be formed

2. Name Search + Formation Prep
- capture proposed legal name
- link to state formation/search resources
- capture formation status:
  - not started
  - name selected
  - filed
  - approved
- attach formation documents when available

3. EIN Setup
- guide user to apply for EIN
- track EIN status:
  - not started
  - submitted
  - assigned
- allow secure storage of EIN confirmation

4. Banking Readiness
- checklist for:
  - EIN received
  - formation approved
  - bank selected
  - account opened
  - authorized signer confirmed
- store banking setup status

5. Tax / Permit Setup
- seller’s permit / tax registration tracking where applicable
- payroll account tracking
- state tax account tracking
- business license tracking
- local permit tracking

6. Automotive Regulatory Setup
- automotive repair licensing / registration planning
- smog / station / related licensing planning if applicable
- ownership change implications
- cannot-operate warnings if licenses are not active

7. Insurance / Employment Readiness
- general liability
- garage liability
- workers’ comp
- payroll setup
- employee onboarding preparation

8. Governance / Legal Docs
- operating agreement or bylaws
- resolutions / member consent
- authority to acquire assets
- signature authority confirmation

9. Readiness Gate
The app should calculate:
- Entity Ready %
- Compliance Ready %
- Day 1 Legal Readiness %
- Close Readiness %

The user should not be shown as “Ready to Close” if the entity setup is incomplete.
