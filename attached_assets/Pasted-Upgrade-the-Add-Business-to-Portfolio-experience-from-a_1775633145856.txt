Upgrade the “Add Business to Portfolio” experience from a basic form into an intelligent intake and workflow-orchestration system.

This should no longer behave like a simple record entry page.

It should behave like an AI-assisted business intake wizard that:
1. captures a more complete business/deal profile
2. infers the likely workflow path
3. adapts buyer and seller workflows automatically
4. identifies missing next steps
5. guides the user forward like an acquisition assistant

==================================================
PRIMARY OBJECTIVE
==================================================

When a user adds a business to the portfolio, the app should gather enough structured context to intelligently configure:
- deal stage
- acquisition workflow
- buyer workflow
- seller workflow
- financing path
- document requests
- risk flags
- next-step recommendations
- missing-step prompts

The result should feel like:
“The app understands this deal and is helping me move it forward.”

==================================================
INTELLIGENT WORKFLOW ADAPTATION
==================================================

After business intake is submitted, the app should infer and configure workflow logic automatically.

The system should create:
- lifecycle stage
- workflow path
- financing path
- buyer action queue
- seller request queue
- document checklist
- missing-item prompts
- risk alerts
- recommended next step

Examples of workflow adaptation:

If stage = Lead:
- create lead workspace
- recommend NDA / initial financial request if not yet received
- keep seller portal minimal
- show “advance to active deal” criteria

If seller has already provided some financials:
- mark preliminary diligence started
- prompt user to upload received financials
- open financial review workflow early
- suggest which missing financial documents are still needed

If SBA financing = yes:
- activate SBA readiness track
- create checklist for lender package
- do not show SBA submission ready until required documents are complete
- surface lender-focused missing items

If acquisition type = Asset Purchase:
- activate New Entity Setup path
- activate transfer/assignment tasks
- deactivate stock-specific tasks

If acquisition type = Stock Purchase:
- activate corporate record review
- activate liability / tax / entity continuity review

If stage = Operating:
- route workspace default home to Financial Management dashboard

AI ASSISTANT BEHAVIOR
==================================================

Add an “AI Deal Assistant” panel to the intake confirmation and business workspace overview.

This assistant should use intake data to generate:

1. Deal Summary
Example:
“Prestige Auto Works is currently in Lead stage, likely SBA-financed, with preliminary financials received but upload and review still pending.”

2. Recommended Next Steps
Example:
- Upload seller-provided financials
- Send missing document request
- Start SBA lender checklist
- Clarify transaction structure
- Confirm lease and landlord path

3. Missing Critical Items
Example:
- Tax returns not received
- Lease not received
- Bank statements not received
- Debt schedule not received

4. Workflow Suggestions
Example:
- Because SBA is likely, activate lender-readiness workflow
- Because financials were partially received, start diligence before LOI refinement
- Because this is still a Lead, seller portal should stay document-light until deal advances

5. Readiness Signals
Example:
- Diligence readiness: low
- Financing readiness: low
- Seller response readiness: medium
- Close readiness: not started

This should feel like an in-app operating assistant, not just static text.
